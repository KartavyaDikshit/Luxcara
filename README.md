# Luxcara AI Concepts Demo

Welcome to the **Luxcara AI Concepts Demo** repository! This project serves as an interactive educational walkthrough of advanced AI engineering concepts, tailored for enterprise and legal workflows. 

The primary demo is located in the `ai_concepts_demo` directory and is fully self-contained using **mock data and simulated API responses**. This means you do **not** need an OpenAI API key or any external service accounts to run these examples.

## 📂 Project Structure

The demo is split into 7 key modules, each containing a Jupyter Notebook (`.ipynb`) that mixes theoretical explanations with executable Python code:

1. **`1_RAG_Mastery`**  
   Deep dive into the Retrieval-Augmented Generation (RAG) pipeline. Covers chunking strategies, dense embeddings, vector retrieval, and context injection.
   
2. **`2_LLM_API`**  
   Production-level API integration techniques. Demonstrates simulated token streaming, token counting, budgeting, error handling with backoff retries, and generation parameters.

3. **`3_Prompt_Engineering`**  
   Advanced prompting frameworks including Zero-Shot, Few-Shot, Chain-of-Thought (CoT) reasoning, and enforcing structured JSON outputs.

4. **`4_Agents`**  
   Building autonomous systems. Showcases a ReAct (Reason + Act) loop, tool registration, and state memory across agent interactions.

5. **`5_Risk_Evaluation`**  
   Safety and compliance guardrails. Implements prompt injection detection, hallucination scoring, and toxicity analysis.

6. **`6_German_Language`**  
   Tackling German-specific AI challenges. Handles compound word splitting, formal vs. informal addressing (*Sie* vs *Du*), and NLP quirks for German documents.

7. **`7_Legal_Walkthrough`**  
   A practical application mapped to compliance workflows, featuring EU AI Act risk classification, audit log generation, and compliance checklists.

## 🚀 How to Run

1. Open the repository in your favorite IDE (like VS Code) or launch Jupyter Lab.
2. Navigate into the `ai_concepts_demo/` folder.
3. Open any of the `.ipynb` notebooks in the subdirectories.
4. Run the cells from top to bottom. Because the notebooks use mock data, they will execute immediately without needing environment variables or API keys.
